# 2026-03-14 Public Release Refactor Log

This change log records the public-facing cleanup completed for the AgentCore OS open-source surface on 2026-03-14.

It is intended to explain one specific round of repository shaping work:

- public-surface split
- runtime naming migration
- release-boundary tightening
- compatibility-preserving cleanup

For the standing policy and release-boundary guidance behind this work, see:

- [Public Release Guide](../PUBLIC_RELEASE.md)

## Why this work was needed

The public repository had reached a point where several historical internal names and release-boundary assumptions were still visible in the codebase.

That created three problems:

1. The public product surface still exposed legacy `openclaw` naming in places where the public-facing product should read as AgentCore OS / Agent Runtime.
2. The public API tree and UI naming did not fully match the intended product language presented to users.
3. The repository boundary still needed clearer protection against accidental inclusion of local operator state, runtime artifacts, and private working files.

This refactor addresses those issues without trying to rewrite every internal compatibility layer at once.

## Main outcomes

### 1. Public runtime naming moved away from legacy OpenClaw labels

The public-facing surface was cleaned up so the repository and product presentation better align around:

- AgentCore OS
- Agent Runtime
- Runtime Console
- `agentcore:*` / `agentcore.*` naming in frontend-facing namespaces

This reduces leakage of historical private/internal terminology in the open-source edition.

### 2. Public API entrypoints were moved to `src/app/api/runtime/**`

The public API path set was reshaped from legacy:

- `src/app/api/openclaw/**`

Toward public runtime entrypoints under:

- `src/app/api/runtime/**`

This makes the public repository structure more consistent with the product language shown in documentation, UI labels, and public packaging.

Representative runtime-side routes in this refactor include:

- `src/app/api/runtime/agent/route.ts`
- `src/app/api/runtime/copy/route.ts`
- `src/app/api/runtime/execute/route.ts`
- `src/app/api/runtime/gateway/health/route.ts`
- `src/app/api/runtime/test/route.ts`
- `src/app/api/runtime/vault/query/route.ts`
- `src/app/api/runtime/assets/[name]/route.ts`
- `src/app/api/runtime/remote-validate/route.ts`
- `src/app/api/runtime/remote-validate/archive/route.ts`

### 3. Public release boundaries were tightened

The release boundary for the public repository was clarified so local/private runtime material is less likely to leak into the open-source tree or archives.

Areas explicitly treated as outside the public release surface include:

- `.openclaw/`
- `.openclaw-data/`
- local operator files
- temporary runtime directories
- machine-specific state
- private validation artifacts

This boundary tightening is part documentation, part repository hygiene, and part packaging discipline.

### 4. Frontend namespace migration was made safer

The frontend-facing naming surface was updated toward:

- `agentcore:*`
- `agentcore.*`

At the same time, compatibility handling was retained so existing local state using legacy `openclaw.*` names can still migrate forward instead of being silently lost.

### 5. Public settings semantics were cleaned up

Public configuration semantics moved away from older `openclaw` wording and toward `runtimeAgent`-style naming where appropriate.

This makes the public config model easier to understand for new users who do not have the historical internal context.

### 6. Compatibility boundaries were intentionally preserved

This refactor is not a hard break.

Compatibility was intentionally preserved in selected places, including:

- `.openclaw-data` local data directory compatibility
- `OPENCLAW_BIN` environment variable compatibility
- selected legacy request-header compatibility
- migration handling for legacy browser/local storage state

That balance keeps the public surface cleaner while avoiding unnecessary disruption for existing local runtime setups.

## Public summary

If you need a short public-facing summary for release notes, PR descriptions, or external changelogs, use the following:

- Renamed the public product surface from legacy **OpenClaw** naming to **Agent Runtime / Runtime Console**, reducing exposure of historical private/internal terminology.
- Migrated public API entrypoints from `src/app/api/openclaw/**` to `src/app/api/runtime/**`, and removed the old public-facing route files.
- Added stricter public release boundaries so `.openclaw/`, `.openclaw-data/`, local operator files, and temporary runtime artifacts do not enter the public repo or release archives.
- Migrated frontend event and local-storage namespaces to `agentcore:*` / `agentcore.*`, while preserving automatic migration from legacy `openclaw.*` local state to avoid local data loss.
- Updated public settings semantics from `openclaw` to `runtimeAgent`, so the exposed configuration model better matches AgentCore OS naming.
- Preserved compatibility boundaries, including `.openclaw-data`, `OPENCLAW_BIN`, and selected legacy request headers, so existing local runtimes do not immediately break.
