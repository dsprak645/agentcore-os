# v0.1.0

Initial open-source release of **OpenClaw OS — Web Desktop UI Framework**.

## Release summary

This release opens the project as an early, usable browser-based desktop UI framework built with Next.js and Tailwind.
It includes the core shell, example workflow apps, local-first state management, and a safe-by-default connector model for publishing flows.

## Highlights

- Desktop-style window manager
  - focus, z-order, minimize/restore, drag/snap, position persistence
- Spotlight launcher
  - app search + command execution
- Local-first data model
  - settings, tasks, drafts, publish history
- Publishing hub
  - safe-by-default dispatch + bring-your-own webhook connectors
- Local webhook connector example
  - receipt logging UI for testing and integration work
- OpenClaw-oriented server routes
  - optional integration points for external generation / workflow execution

## Important notes

- This is an **initial foundation release**, not a hardened production platform.
- Secrets are **not securely stored by default** in this demo architecture.
- Any “auto publish” capability should be implemented only through:
  - official platform APIs,
  - approved third-party services, or
  - internal tooling with explicit user consent and policy compliance.

## Recommended next steps for users

1. Run the app locally with `npm install && npm run dev`
2. Review architecture and privacy docs before customization
3. Replace local-secret patterns before any multi-user or internet-facing deployment
4. Keep connector implementations compliant and auditable
