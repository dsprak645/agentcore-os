# AgentCore OS v0.2.0-beta.2

`v0.2.0-beta.2` is a limited technical preview release in the early AgentCore OS beta line.

It is kept public mainly for chronology, implementation reference, and packaging verification, rather than as the primary recommended public entry point for general users.

## Positioning

Compared with `v0.2.0-beta.1`, this release should be understood as a transitional build that captures follow-up workspace, settings, documentation, and desktop-runtime iteration accumulated on `main`.

For a cleaner understanding of the public project direction, naming transition, and product framing, `v0.2.0-beta.1` remains the better primary reference point.

## What this release primarily represents

### Workspace and desktop iteration

- Reorganized the desktop surface around work categories instead of a single flat app grid.
- Shared category metadata now drives names, descriptions, and placement hints across both the desktop and Settings.
- Desktop overview cards now show category counts and correct Dock totals.
- Workspace presets now cover creator, sales, support, research, operations, personal, and language-learning scenarios.
- Industry App Center continues to support industry bundles, scenario-based setup, and custom workspace composition.
- Solutions Hub more clearly maps mature use cases into installable workspace/app flows.

### Settings and personalization follow-up

- Redesigned `App 配备` from plain checkbox rows into category cards with bulk actions.
- Added clearer Desktop / Dock counts and per-category quick actions.
- Custom workspace initialization now starts from the selected scenario when appropriate.
- Explicitly empty Desktop or Dock selections are now preserved instead of silently restoring defaults.

### Session continuity and UX fixes

- Session history now restores automatically from local storage.
- Orphaned message history can be recovered back into visible sessions.
- Retention was expanded to support more sessions and more preserved messages.
- The session strip now supports longer history navigation and a quick `回到最近` action.
- Fixed a session-isolation issue where in-flight replies could land in the wrong conversation after switching sessions.

### Documentation and packaging verification

- Expanded `docs/GETTING_STARTED.md` into practical setup tracks for browser-only, desktop shell + local sidecar, and desktop shell + `lobster-src` compatibility work.
- Expanded `docs/CONFIGURATION.md` with staged recommendations for LLM setup, publish/webhook flows, desktop runtime features, and LobsterAI-related integration work.
- Documentation now separates Node, Python, Rust, and Lobster runtime expectations more clearly.
- The repo’s Tauri desktop packaging path is verified on both macOS and Windows for this release snapshot.
