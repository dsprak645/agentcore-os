# AgentCore OS v0.2.0-beta.1

`v0.2.0-beta.1` is the first public beta release under the **AgentCore OS** name.

This release marks the project’s transition from the earlier OpenClaw framing toward a lower-friction, privacy-first local AI productivity foundation that can do real work and steadily accumulate operational assets.
It is aimed not only at technical users, but also at business owners, creators, beginners, and operators who need a safer and more practical way to run AI in real workflows.

## Rename announcement

This release marks an important milestone for the project.

As the system has evolved through deeper architectural changes and a major shift in product direction, the original name **OpenClaw** no longer fully reflects what this project is becoming. The project is now officially renamed to **AgentCore OS**.

### Why the rename?

Over the last iterations, one insight became increasingly clear: users do not actually need a larger pile of AI apps. What they need is a system that can directly carry real business workflows.

That realization led to a deeper restructuring of the product architecture.

Apps are no longer the center of the product story. They are becoming execution components underneath a stronger system spine built around:

- **Agents** for goal-oriented execution
- **Core** for workflow context, role surfaces, and business-state coordination

That is why **AgentCore OS** is now a more accurate name for the project than OpenClaw OS.

### What has fundamentally changed?

#### 1. Role Desks

The product is moving beyond a generic desktop surface and toward role-oriented workspaces such as **Creator Desk** and **Sales Desk**, giving users a more direct and immediately usable starting point.

#### 2. Workflow Runtime with state

Apps are no longer loosely connected. Core flows are now increasingly organized around a stateful runtime using `workflowRunId`, allowing context, steps, and outputs to stay connected across the workflow.

#### 3. Asset Console

Results are no longer disposable. Workflow outputs can now be retained as local-first business assets, with deeper continuity between execution, storage, and re-entry into the workflow.

## Compatibility

No breaking changes planned for existing preview users, though this remains a beta release and implementation details may still evolve.

## Known limits

- This is still a pre-release branch state, not a hardened production release.
- Some OpenClaw-dependent health checks may depend on local gateway/runtime availability.
- Secrets and settings are still local-first demo behavior unless replaced in deployment.
