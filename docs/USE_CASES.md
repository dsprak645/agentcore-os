# Use Cases (Solo / Small Teams)

This project is a **desktop-style UI shell** plus a set of workflow primitives. Below are common scenarios you can run with the current app set.

## Quick path: install a workflow pack

Open **方案库 / Solutions Hub** and install a pack. It will add a set of runnable SOPs into **SoloOps → 我的 Playbooks** (local-first).

## 1) Content pipeline (idea → publish → iterate)

- Use **Workflow Playbooks** to follow a repeatable SOP
- Generate variants in **Media Ops** and assets in **Creative Studio**
- Use **Publisher** for “dry-run” payload previews, then dispatch to your connector
- Capture what worked in **Knowledge Vault** as reusable templates/snippets

### Recommended packs (Solutions Hub)

- 内容生产流水线（从选题到复盘）
- Newsletter / 周报系统（收集 → 写作 → 分发）
- 品牌素材与一致性（Brand Kit）

## 2) Repurposing (one source → many formats)

- Long-form → short posts (per platform constraints)
- Video → cover + highlight clip + caption set
- Weekly summary → newsletter + carousel + short scripts

## 3) Customer support scripts (comments / DMs / tickets)

- Maintain a response library in **Knowledge Vault**
- Use **Spotlight** to run quick “reply generation” commands
- Optionally route inbound messages through your connector to create tasks/drafts

### Recommended packs (Solutions Hub)

- 评论区 / 私信 / 工单（标准回复 + 跟进）

## 4) Product launch checklist (solo founder-friendly)

- Use **Task Manager** for launch timelines and checklists
- Store pricing/positioning/FAQ in **Knowledge Vault**
- Produce launch content packs and dispatch via **Publisher**

### Recommended packs (Solutions Hub)

- 产品/活动发布（从准备到上线）
- 电商上新与促销（素材包 + 发布节奏）

## 5) Storefront ops (lightweight)

- Keep promo calendars, product notes, and creative assets organized
- Track publish history and iterate on messaging over time

## More “mature” scenarios (Solutions Hub)

- 线索获取漏斗（内容 → 表单 → 跟进）
- SaaS 增长闭环（获客 → 激活 → 留存）
- 课程/知识产品（内容引流 → 转化 → 交付）
- B2B 外联系统（名单 → 触达 → 跟进）
- 招聘/外包交付（需求 → 筛选 → 交付验收）
- 日常运营节奏（每日清单 + 周复盘）

## Notes on automation

OpenClaw OS intentionally does **not** ship platform-bypass automation. For “auto publish”, use:

- Official APIs, or
- Approved tools (Make/Zapier/Buffer/Metricool), or
- Your own internal tooling with explicit user consent and ToS compliance
